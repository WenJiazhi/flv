FLV Proxy Player

粘贴完整请求头，直接代理播放直播

浏览器本身不能稳定伪造完整请求头，这个页面会先把你的请求头交给本地 Node 服务，再由服务端按同样的 header 去拉取 FLV 直播流并转发给播放器。遇到浏览器不支持的 FLV 编码时，会自动切到 ffmpeg 转码兼容模式。

支持的输入

Chrome DevTools 里复制出的请求头文本

完整请求首行 + Host + Cookie + Referer

HTTP/2 伪头风格：`:method` / `:scheme` / `:authority` / `:path`

兼容说明

`flv.js` 只能直接播放浏览器支持的编码，常见是 H.264/AAC。

如果上游 FLV 里是 HEVC/增强 FLV，自动模式会切换到服务端转码。

转码会明显增加 CPU 占用，但能解决大多数“codec unsupported”问题。
